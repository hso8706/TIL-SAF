### AJAX
- 화면 갱신 없이 비동기 방식으로 데이터를 서버에서 가져와 처리하는 방법
- 예시
  - 회원 가입 중복 체크를 비동기 방식으로 해결 가능

### 비동기 처리를 위한 방식
1. XMLHttpRequest 사용(XHR)
   - 가장 기본적인 방법
2. fetch()
3. axious

### Ajax 통신 원리
- 폼 전송 방식이 아닌 자바스크립트 객체(XMLHttpRequest, XHR)를 이용한 요청 및 응답을 처리하는 원리
- 웹 브라우저가 비동기 요청을 받으면 콜백 함수를 실행
  - 콜백 함수
    - 특정 조건이 되면 실행되게함.

### Ajax 실행 process
1. 이벤트 발생
2. XHR 객체 생성
3. Client에서 웹 서버로 요청 전송
4. 웹 서버에서 응답 생성
5. 웹 서버에서 Client로 응답 전송
6. 응답 처리 메서드 실행

### XHR 객체의 속성
- **readyState**
  - AJAX 객체의 요청, 응답의 상태를 나타내는 숫자(0 ~ 4)
    - 0: request가 초기화 되지 않음 (open() 호출 전)
    - 1: request가 셋업되고 보내지 않음 (send() 호출 전)
    - 2: requset가 보내지고, 진행 중 일 경우 (send() 호출 후)
    - 3: response에서 부분적인 데이터를 받으며, 완료가 아직 안 된 경우
    - 4: 서버의 응답이 완료된 상태
  - 4번이 아닌 경우를 로딩중으로 처리할 수 있음
- **onreadystatechange**
  - 요청에 대한 응답을 받을때마다, 즉 readyState 속성(0 ~ 4)이 바뀔 때마다 호출됨
  - 콜백 함수 지정 속성
- **status**
  - 서버로부터 응답된 HTTP 상태 코드를 나타내는 숫자
  - 200 : 정상적으로 응답을 받은 경우
  - 404 : 페이지를 찾지 못한 경우
  - 405 : 요청 방식이 다름
  - 500 : 서버 오류 발생
- 응답 종류
  - statusText
  - responseText
    - JSON 은 우선 Text로 받음
  - responseXMl

### fetch()
- 브라우저에서 제공해주는 함수
- 첫번쨰 인자 : URL
- 두번쨰 인자 : options 객체
  - options의 디폴트 값은 get 방식으로 진행됨
- 리턴 타입 : Promise 타입
  - 반환된 Promise 객체는 API 호출이 성공했을 경우, response 객체를 resolve한다.
  - 반환된 Promise 객체는 API 호출이 실패했을 경우, 예외(error) 객체를 reject한다.

### fetch().then()
- then()
  - fetch() 함수가 성공적으로 호출된 경우 then() 함수가 실행된다.
  - then()은 체인 형식으로 여러개를 엮을 수 있음
