### ddl-auto 옵션
- create: 애플리케이션 실행 시점에 기존 테이블을 삭제하고 새로운 테이블을 생성합니다. 데이터가 초기화됩니다.
- create-drop: 애플리케이션을 종료할 때 생성된 테이블을 삭제합니다.
- update: 애플리케이션 실행 시점에 스키마를 업데이트하여 변경된 부분만 적용합니다. 기존 데이터는 유지됩니다.
- validate: 애플리케이션 실행 시점에 데이터베이스와 엔티티 클래스의 스키마를 검증합니다. 변경된 부분은 적용되지 않습니다.
- none: 스키마 자동 생성을 사용하지 않습니다.

### 개발 환경과 배포 환경
- 개발 환경 : 주로 update
- 배포 환경 : 주로 none
