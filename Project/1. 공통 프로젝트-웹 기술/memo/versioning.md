# Versioning

### 버저닝 구조
    <major>.<minor>.<etc>

    - major
      - 호환이 안되는 변경, Framework 변경, 함수 삭제, 이름 변경 등의 커다란 변경사항
      - 구조 자체가 변화
      - 아예 다른 프로그램이 되는 느낌같다.
    - minor 
      - 호환이 가능한 변경, 기능 추가, 컴포넌트 추가, 클래스 추가, 함수 추가 등, 변경사항이 이전의 버전에서 추가되는 것
    - etc : 버그 수정, 약간의 디자인 변경, 사소한 변동사항 등을 의미

### 0의 의미
    - 0.x
      - major은 0부터 시작할 수 있음.
      - 0은 개발 중이라는 의미. 혹은 서비스 운영 중이어도 사용법이 아예 바뀌는 변화가 있을 수 있다, 즉 불안정하다는 의미

### 문제점
    major, minor, etc에 대한 기능 설정이 주관적이다.
